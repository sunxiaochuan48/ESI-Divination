## 这是什么

**ESI-Divination** 是一个以实证（Empirical）、系统（Systematic）、迭代（Iterative）为核心指导，针对术数占卜的实用性与原理进行探究的开源项目。

在当前阶段，我们主要以 AI Agent Skill 进行项目展开。

我们探究的是：符号经过排列组合后能否真实的反映现实。六爻等符号体系只是我们在权衡后的一个选择，一个实验载体。在项目落地过程中，我们会尽可能的去除神秘化元素，我们也在 AI Agent Skill 中对巴纳姆效应进行了禁用，这会证明，它不靠话术，错就是错。

## 三大核心哲学

### 01 实证

实践为证，不做理论与精神上的"巨人"，优先以实践建立认知并以此为方针。从现象中得到的稳定反馈，比"理论正确"和"道路正确"更重要。每一次占卜都基于可观测的卦象和可复现的起卦过程——不做猜测驱动，而是证据驱动。

### 02 系统

让 AI 按照自己的度量衡来进行猜想，远不如建立一套系统的架构，让每一句判断都能追溯到源头，让解读变成一个又一个可编辑优化的结构。结构先于直觉——体系化的排盘规则、标准化的解卦流程、可度量的验证指标，让玄学变得可管理、可学习。

### 03 迭代

我们不满足于单一版本，不满足于做过就好，让它变成一个玩具——这绝非我们的目的。通过测试不断发觉缺陷并反馈优化，不断迭代，才能带来真正的未来。完美不是终点，持续精进才是，让每一次解卦都比上一次更通透。

## 项目目标

| 目标 | 说明 |
|:----:|------|
| **G1** | **术数占卜的实际落地**——以 AI Agent Skill 为载体，将术数占卜流程转化为可执行、可复用的标准化操作步骤，以实用性为首要目标，这也是整个项目的核心关注点。 |
| **G2** | **客观有效的测试框架**——设计并搭建可量化、可复现的测试框架，通过客观指标评估术数占卜在测试环境中的有效性。AI Agent Skill 将作为测试执行的主体，在多元与二元测试题目中运行，得出多个参考正确率指标。 |
| **G3** | **反馈驱动的稳定迭代**——系统性收集每一轮测试中的误差与不足，融入下一代 Skill 的改进中，形成"测试—反馈—优化"的闭环机制，推动参考正确率在迭代中持续精进。 |
| **G4** | **理论层面的探索**——在实践积累的基础上，尝试对术数体系的内在逻辑进行探究，以期形成更具启发性的理解。这些论点源于对现象的观察与归纳总结。 |
